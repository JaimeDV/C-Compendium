\doxysubsection{Detailed Description}
Features that implement in C++ the GLSL specification as closely as possible. 

The core of GLM, which implements exactly and only the GLSL specification to the degree possible.

The GLM core consists of C++ types that mirror GLSL types and C++ functions that mirror the GLSL functions.

The best documentation for GLM Core is the current GLSL specification, \href{http://www.opengl.org/registry/doc/GLSLangSpec.4.20.8.clean.pdf}{\texttt{ version 4.\+2 (pdf file)}}.

GLM core functionalities require \texorpdfstring{$<$}{<}glm/glm.\+hpp\texorpdfstring{$>$}{>} to be included to be used.

The GLM core consists of \doxylink{group__core__types}{C++ types that mirror GLSL types} and C++ functions that mirror the GLSL functions. It also includes \doxylink{group__core__precision}{a set of precision-\/based types} that can be used in the appropriate functions. The C++ types are all based on a basic set of \doxylink{group__core__template}{template types}.

The best documentation for GLM Core is the current GLSL specification, \href{http://www.opengl.org/registry/doc/GLSLangSpec.4.20.8.clean.pdf}{\texttt{ version 4.\+2 (pdf file)}}.

GLM core functionnalities require \texorpdfstring{$<$}{<}glm/glm.\+hpp\texorpdfstring{$>$}{>} to be included to be used.
