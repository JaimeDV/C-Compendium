Get a snapshot of the current state of the keyboard.

The pointer returned is a pointer to an internal SDL array. It will be valid for the whole lifetime of the application and should not be freed by the caller.

A array element with a value of 1 means that the key is pressed and a value of 0 means that it is not. Indexes into this array are obtained by using SDL\+\_\+\+Scancode values.

Use \doxylink{_s_d_l__events_8h_aa15403d1c8fc341cd9cf750e27a66d19}{SDL\+\_\+\+Pump\+Events()} to update the state array.

This function gives you the current state after all events have been processed, so if a key or button has been pressed and released before you process events, then the pressed state will never show up in the \doxylink{_s_d_l__keyboard_8h_a6df72a92c606bfcf9ef00dc1b049270f}{SDL\+\_\+\+Get\+Keyboard\+State()} calls.

Note\+: This function doesn\textquotesingle{}t take into account whether shift has been pressed or not.

To start text input in a given location, this function is intended to be called before SDL\+\_\+\+Start\+Text\+Input, although some platforms support moving the rectangle even while text input (and a composition) is active.

Note\+: If you want to use the system native IME window, try setting hint {\bfseries{SDL\+\_\+\+HINT\+\_\+\+IME\+\_\+\+SHOW\+\_\+\+UI}} to {\bfseries{1}}, otherwise this function won\textquotesingle{}t give you any feedback.

Start accepting Unicode text input events.

This function will start accepting Unicode text input events in the focused SDL window, and start emitting \doxylink{struct_s_d_l___text_input_event}{SDL\+\_\+\+Text\+Input\+Event} (SDL\+\_\+\+TEXTINPUT) and \doxylink{struct_s_d_l___text_editing_event}{SDL\+\_\+\+Text\+Editing\+Event} (SDL\+\_\+\+TEXTEDITING) events. Please use this function in pair with \doxylink{_s_d_l__keyboard_8h_ab08b914cdfb2eaed26165cb2f6b110ec}{SDL\+\_\+\+Stop\+Text\+Input()}.

On some platforms using this function activates the screen keyboard.
