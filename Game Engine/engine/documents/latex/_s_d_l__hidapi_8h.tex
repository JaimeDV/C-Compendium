Check to see if devices may have been added or removed.

Enumerating the HID devices is an expensive operation, so you can call this to see if there have been any system device changes since the last call to this function. A change in the counter returned doesn\textquotesingle{}t necessarily mean that anything has changed, but you can call \doxylink{_s_d_l__hidapi_8h_a5a0a8a98f734167ad2272c0152f51c92}{SDL\+\_\+hid\+\_\+enumerate()} to get an updated device list.

Calling this function for the first time may cause a thread or other system resource to be allocated to track device change notifications.

\begin{DoxyReturn}{Returns}
a change counter that is incremented with each potential device change, or 0 if device change detection isn\textquotesingle{}t available.
\end{DoxyReturn}

Send a Feature report to the device.

Feature reports are sent over the Control endpoint as a Set\+\_\+\+Report transfer. The first byte of {\ttfamily data} must contain the Report ID. For devices which only support a single report, this must be set to 0x0. The remaining bytes contain the report data. Since the Report ID is mandatory, calls to \doxylink{_s_d_l__hidapi_8h_a5349c62d24338025552c6f244956699f}{SDL\+\_\+hid\+\_\+send\+\_\+feature\+\_\+report()} will always contain one more byte than the report contains. For example, if a hid report is 16 bytes long, 17 bytes must be passed to \doxylink{_s_d_l__hidapi_8h_a5349c62d24338025552c6f244956699f}{SDL\+\_\+hid\+\_\+send\+\_\+feature\+\_\+report()}\+: the Report ID (or 0x0, for devices which do not use numbered reports), followed by the report data (16 bytes). In this example, the length passed in would be 17.
