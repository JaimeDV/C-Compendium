 The spine-\/sfml runtime provides functionality to load, manipulate and render \href{http://esotericsoftware.com}{\texttt{ Spine}} skeletal animation data using \href{http://www.sfml-dev.org/}{\texttt{ SFML}}. spine-\/sfml is based on \href{https://github.com/EsotericSoftware/spine-runtimes/tree/master/spine-c}{\texttt{ spine-\/c}}.\hypertarget{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2spine-sfml_2_r_e_a_d_m_e_autotoc_md306}{}\doxysection{\texorpdfstring{Licensing}{Licensing}}\label{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2spine-sfml_2_r_e_a_d_m_e_autotoc_md306}
This Spine Runtime may only be used for personal or internal use, typically to evaluate Spine before purchasing. If you would like to incorporate a Spine Runtime into your applications, distribute software containing a Spine Runtime, or modify a Spine Runtime, then you will need a valid \href{https://esotericsoftware.com/spine-purchase}{\texttt{ Spine license}}. Please see the \href{https://github.com/EsotericSoftware/spine-runtimes/blob/master/LICENSE}{\texttt{ Spine Runtimes Software License}} for detailed information.

The Spine Runtimes are developed with the intent to be used with data exported from Spine. \item Download the Spine Runtimes source using \href{https://help.github.com/articles/set-up-git}{\texttt{ git}} or by downloading it \href{https://github.com/EsotericSoftware/spine-runtimes/archive/master.zip}{\texttt{ as a zip}}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Using Eclipse CDT, import the project by choosing File -\/\texorpdfstring{$>$}{>} Import -\/\texorpdfstring{$>$}{>} Existing projects. For other IDEs you will need to create a new project and import the source.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Copy the SFML binaries into the {\ttfamily spine-\/sfml/\+Debug} directory so they can be found when the example is run.
\end{DoxyEnumerate}

Alternatively, the contents of the {\ttfamily spine-\/c/src}, {\ttfamily spine-\/c/include} and {\ttfamily spine-\/sfml/src} directories can be copied into your project. Be sure your header search path will find the contents of the {\ttfamily spine-\/c/include} and {\ttfamily spine-\/sfml/src} directories.
