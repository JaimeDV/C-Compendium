\doxysubsection{Detailed Description}
\doxylink{classbt_constraint_solver_pool_mt}{bt\+Constraint\+Solver\+Pool\+Mt} -\/ masquerades as a constraint solver, but really it is a threadsafe pool of them.

Each solver in the pool is protected by a mutex. When solve\+Group is called from a thread, the pool looks for a solver that isn\textquotesingle{}t being used by another thread, locks it, and dispatches the call to the solver. So long as there are at least as many solvers as there are hardware threads, it should never need to spin wait.
