\item Contact callbacks\+: begin, end, pre-\/solve, post-\/solve
\item Convex polygons and circles
\item Multiple shapes per body
\item One-\/shot contact manifolds
\item Dynamic tree broadphase
\item Efficient pair management
\item Fast broadphase AABB queries
\item Collision groups and categories
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md5}{}\doxysubsubsection{\texorpdfstring{Physics}{Physics}}\label{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md5}

\begin{DoxyItemize}
\item Continuous physics with time of impact solver
\item Persistent body-\/joint-\/contact graph
\item Island solution and sleep management
\item Contact, friction, and restitution
\item Stable stacking with a linear-\/time solver
\item Revolute, prismatic, distance, pulley, gear, mouse joint, and other joint types
\item Joint limits, motors, and friction
\item Momentum decoupled position correction
\item Fairly accurate reaction forces/impulses

\item Install \href{https://cmake.org/}{\texttt{ CMake}}
\item Ensure CMake is in the user {\ttfamily PATH}
\item Visual Studio\+: run {\ttfamily build.\+bat} from the command prompt
\item Otherwise\+: run {\ttfamily build.\+sh} from a bash shell
\item Results are in the build sub-\/folder
\item On Windows you can open box2d.\+sln

You can download and install Box2D using the \href{https://github.com/Microsoft/vcpkg}{\texttt{ vcpkg}} dependency manager\+:


\begin{DoxyItemize}
\item git clone \href{https://github.com/Microsoft/vcpkg.git}{\texttt{ https\+://github.\+com/\+Microsoft/vcpkg.\+git}}
\item cd vcpkg
\item ./bootstrap-\/vcpkg.sh
\item ./vcpkg integrate install
\item ./vcpkg install box2d
\end{DoxyItemize}

The Box2D port in vcpkg is kept up to date by Microsoft team members and community contributors. If the version is out of date, please \href{https://github.com/Microsoft/vcpkg}{\texttt{ create an issue or pull request}} on the vcpkg repository.

Note\+: vcpkg support is not provided by the Box2D project\hypertarget{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md10}{}\doxysubsection{\texorpdfstring{Building for Xcode}{Building for Xcode}}\label{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md10}

\begin{DoxyItemize}
\item Install \href{https://cmake.org}{\texttt{ CMake}}
\item Add Cmake to the path in .zprofile (the default Terminal shell is zsh)
\begin{DoxyItemize}
\item export PATH="{}/\+Applications/\+CMake.\+app/\+Contents/bin\+:\$\+PATH"{}
\end{DoxyItemize}
\item mkdir build
\item cd build
\item cmake -\/G Xcode ..
\item open box2d.\+xcodeproj
\item Select the testbed scheme
\item Edit the scheme to set a custom working directory, make this be in box2d/testbed
\item You can now build and run the testbed
\end{DoxyItemize}\hypertarget{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md11}{}\doxysubsection{\texorpdfstring{Installing using CMake}{Installing using CMake}}\label{md__c_1_2_users_2_usuario_2_documents_2pruebas-c_2_practica_01_animaci_x_c3_x_b3n_2libraries_2box2d-2_84_81_2_r_e_a_d_m_e_autotoc_md11}
You can use the CMake install feature to deploy the library to a central location that can be accessed using\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{find\_package(box2d\ REQUIRED)}
\DoxyCodeLine{target\_link\_libraries(mytarget\ PRIVATE\ box2d)}

\end{DoxyCode}
 You can build and install the library and docs using this command sequence (requires Doxygen)\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cd\ build}
\DoxyCodeLine{cmake\ -\/DBOX2D\_BUILD\_DOCS=ON\ ..}
\DoxyCodeLine{cmake\ -\/-\/build\ .}
\DoxyCodeLine{cmake\ -\/-\/build\ .\ -\/-\/target\ INSTALL}

\end{DoxyCode}
 On Windows this tries to install in {\ttfamily Program Files} and thus requires admin privileges. Alternatively you can target another directory using something like this\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir\ build}
\DoxyCodeLine{cd\ build}
\DoxyCodeLine{cmake\ -\/DBOX2D\_BUILD\_DOCS=ON\ -\/DCMAKE\_INSTALL\_PREFIX="{}C:/packages"{}\ ..}
\DoxyCodeLine{cmake\ -\/-\/build\ .}
\DoxyCodeLine{cmake\ -\/-\/build\ .\ -\/-\/target\ INSTALL}

\end{DoxyCode}
